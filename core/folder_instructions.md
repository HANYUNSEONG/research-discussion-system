# Folder Instructions Template

## 00_context

Keep this folder concise and durable. It should be readable at the start of every future session.
Keep user-facing preferences in `user_profile.md` so the user can edit them in one place.

## 04_discussions

Write one Markdown file per bounded discussion. Do not use chat transcript dumps. Summarize into structured sections and preserve links to source files.

## 05_literature

Prefer one note per paper or source. Link claims in discussions back to literature notes when possible.

## 99_meta

Do not edit runtime settings casually. If changing linking mode, scaffold, or agent contract, record the change in `scaffold_decisions.md`.
