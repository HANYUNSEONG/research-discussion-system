# RDS New Discussion

Use this skill to start a bounded research discussion.

## Workflow

1. Ask for the topic if not already clear.
2. Create a short working title suitable for a filename.
3. Identify materials to ground the discussion:
   - data files
   - figures
   - protocols
   - literature notes
   - prior discussions
   - open questions
4. Read the relevant files before giving substantive interpretation.
5. During the discussion, separate:
   - direct observations
   - user interpretation
   - AI-generated hypotheses
   - uncertainty
   - suggested next experiments or analyses
6. Keep notes that can later be written into `04_discussions/YYYY-MM-DD_topic.md`.

## Rule

A useful RDS discussion is grounded in files whenever possible. If no files are available, clearly mark the discussion as conceptual.
